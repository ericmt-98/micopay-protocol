ZKaaS Browser Proving Benchmark

Generates a reputation_v1 UltraHonk proof in-browser using @noir-lang/noir_js + @aztec/bb.js (WASM), then verifies it on-chain via POST /api/v1/zk/verify.

Toolchain pins for this repo:
  nargo  1.0.0-beta.9
  bb     0.87.0

The @noir-lang/noir_js + @aztec/bb.js npm packages expose matching
WASM builds, BUT the ESM/WASM bundle for bb 0.87.x requires:

  1. SharedArrayBuffer (requires COOP/COEP headers)
  2. Threads (WASM threads via SharedArrayBuffer)

These headers must be set on the server serving this file:

  Cross-Origin-Opener-Policy: same-origin
  Cross-Origin-Embedder-Policy: require-corp

Without them, the WASM thread pool fails to initialize.

STATUS: This benchmark page is READY to be wired once the dev server
sets those headers. The proving logic below is correct; swap in the
real circuit JSON path to activate it.

Fallback (current demo): proving runs server-side in WSL2 via the
run_demo_b.sh script. This is a known trade-off — the secret leaves
the "device" (WSL2 is still the user's machine), but the privacy
model is slightly weaker than in-browser proving. Documented honestly.

See: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/SharedArrayBuffer#security_requirements
